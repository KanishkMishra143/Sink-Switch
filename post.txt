Switching audio devices shouldn’t break your flow — but it does.

I switch between 3 devices constantly, and the default experience (especially on Linux setups like Hyprland) felt unnecessarily slow and mouse-heavy.

So I built **Sink Switch** — a cross-platform utility to cycle audio outputs instantly using a hotkey.
👉 https://kanishkmishra143.github.io/Sink-Switch

What started as a small Bash wrapper around `pactl` evolved into a deeper systems problem when I tried solving it on Windows.

On Linux:

* Clean interfaces via PulseAudio/PipeWire
* Bash was enough to build a fast, native-feeling solution

On Windows:

* Outdated, restrictive audio APIs
* PowerShell + AutoHotkey approach broke down quickly

I ended up rebuilding the Windows version in Go, directly interfacing with Core Audio APIs for reliable, low-latency switching.

One interesting edge case: tools like FxSound always appear as the active output, even when they’re just routing audio internally — which required tracking actual device state beyond what the API exposes.

A couple of design decisions made a big difference:

* **Cycling > selection** → faster, predictable, no UI friction
* **Config-driven logic** → defines the cycle + maintains state

This wasn’t about building something flashy — just removing friction from a workflow I use every day.

It’s now part of my daily setup, and one of those small tools that quietly improves everything around it.

Built for:

* power users
* multi-device setups
* keyboard-first workflows

Sometimes the best tools come from solving your own problems — and then taking the time to engineer them properly.
